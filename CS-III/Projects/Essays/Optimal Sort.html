Bubble Sort Selection Sort Insertion Sort

In this chart we can see the relation between the number of words sorted and the time it takes to run the program. In all the tests we can see a positive exponential regression as the time taken to perform an algorithm increases wit the number of words used. For the case of bubble sort, the fastest tests were done with the ordered text files. This is because this the most optimal situation for Bubble Sort that total run time is only O(n). However, on average the run time of Bubble Sort is O(n2). Bubble Sort must perform O(n) operations on O(n) objects. Because of this Bubble Sort is an inefficient sorting algorithm for a large list of unsorted objects. Insertion Sort shows a similar exponential regression. Insertion Sort's most optimal scenario is a list of already sorted objects resulting in a run time of O(n). The worst case scenario is a list of reversed order, seen in the charts as those runs reflected the highest run times. The time complexity of this is O(n2). The final algorithm Selection Sort also has a worst case time complexity of O(n2) and a best case of O(n). However, Selection Sort generally performs worse than Insertion Sort. This is also seen in the chart by a exponential regression of the run time. The data of the graphs show how all three of these functions are not optimal for the sorting of large lists as the time complexity is exponential resulting in staggeringly increasing run times.
